docs에 도구 관련 가이드라인을 확인해줘
현재 도구 등록 기능 관련해서 개선 및 신규 기능 추가할거야
현재 AI를 통한 도구 생성을 하면 pre process는 거의 하는 일 없이 bop 전체를 그대로 넘기고, 도구 스크립트에서 그걸 파싱하더라 그래서 해당 문제를 다음 순서로 해결할거야
내가 제안한 방법을 검토하고 보완하여 작업을 수행해 줘
1. py 업로드 기능 + input/output 스키마를 옵션으로 받을거야, 코드 분석만으론 input/output을 정확히 받기 어려우니 스키마도 옵션으로 받고 싶어
2. 도구 생성 시 해당 도구의 스키마와 어댑터만 생성을 먼저 하는 옵션을 만들거야.
도구 자체는 최적화 전문가가 해당 스키마를 참고해서 만들어서 올리는 방법인거야.
3. 도구 생성을 ai로 하는 경우에도 2번을 수행하고 해당 스키마에 맞춰서 기능을 수행하는 도구를 만들거라, 어댑터와 기능이 분리가 잘 될 것 같아.
4. 도구 관련 기능에 대해 클로드 코드가 잘 동작했는지 확인할 수 있게 로깅 부분을 강화해줘. 너무 긴 걸 다 저장하지 말고, 디버깅에 필요한것만 잘 저장하면 좋겠어
5. docs 도구 가이드라인을 작업한 내용에 맞춰 업데이트 해줘
